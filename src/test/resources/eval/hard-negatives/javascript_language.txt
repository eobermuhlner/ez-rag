JavaScript is a lightweight, interpreted or just-in-time compiled programming language with first-class functions. It is most well-known as the scripting language for web pages, but it is also used extensively in server-side, mobile, and desktop environments. JavaScript is the only programming language that runs natively in web browsers without any plugin or compilation step.

JavaScript was created by Brendan Eich in just ten days in May 1995 while working at Netscape Communications. It was initially named Mocha, then LiveScript, before being renamed JavaScript in a marketing partnership with Sun Microsystems to capitalize on the popularity of Java. Despite the similar name, JavaScript and Java are very different languages with fundamentally different designs and use cases.

The ECMAScript specification standardizes JavaScript. The language was standardized by ECMA International as ECMAScript in 1997. ECMAScript 5 (2009) was a significant update that added strict mode and several new methods. ECMAScript 6 (ES6), officially called ES2015, was the most transformative update, introducing arrow functions, classes, template literals, destructuring assignment, default parameters, rest and spread operators, let and const declarations, modules (import/export), Promises, generators, iterators, Map, Set, Symbol, and WeakMap. Since 2015, the specification has been updated annually, with ES2022 adding top-level await and class fields.

JavaScript is prototype-based rather than class-based. Every object in JavaScript has a prototype, and objects inherit directly from other objects rather than from classes. The prototype chain is traversed when looking up properties or methods: if an object does not have a property, its prototype is checked, then the prototype's prototype, and so on up to Object.prototype. The class syntax introduced in ES6 provides a cleaner way to define constructor functions and prototype chains, but it is syntactic sugar over the existing prototype-based system; JavaScript did not add true class-based inheritance.

JavaScript is dynamically typed. Variables do not have a declared type, and a single variable can hold different types of values at different times. JavaScript has seven primitive types: string, number (all numeric values including integers and floats are 64-bit floating point), boolean, null, undefined, Symbol (unique identifiers), and BigInt (arbitrarily large integers). The typeof operator returns a string indicating the type of a value, though famously typeof null returns "object" due to a historical bug that cannot be changed without breaking existing code.

JavaScript's event-driven programming model uses an event loop to handle asynchronous operations without multiple threads. When an asynchronous operation (like a network request or timer) completes, its callback is placed in the event queue. The event loop continuously checks if the call stack is empty and, if so, processes the next item from the event queue. This model allows JavaScript to perform I/O without blocking, making it efficient for network-intensive applications. The microtask queue (for Promise callbacks) has priority over the regular macrotask queue.

Promises were introduced in ES6 to address callback hell (deeply nested callbacks). A Promise represents the eventual completion or failure of an asynchronous operation and its resulting value. Promises can be chained using then() and errors caught with catch(). async and await, introduced in ES2017, build on Promises to provide synchronous-looking syntax for asynchronous code: an async function returns a Promise, and await pauses execution until the awaited Promise resolves. This significantly simplifies writing and reading asynchronous code.

Closures are one of JavaScript's most powerful and frequently used features. A closure is a function that retains access to the variables of its enclosing scope even after the outer function has returned. This is possible because JavaScript functions maintain a reference to the scope in which they were created. Closures are the mechanism behind many JavaScript patterns: module patterns (where private state is maintained), event handlers (which capture state from the enclosing scope), and factory functions (which generate functions with specialized behavior).

Node.js, created by Ryan Dahl in 2009, brought JavaScript to server-side development. Built on Chrome's V8 JavaScript engine, Node.js uses an event-driven, non-blocking I/O model that makes it efficient for building scalable network applications. Node.js enabled the npm ecosystem, which grew to become the world's largest package registry with millions of packages. Popular server-side frameworks built on Node.js include Express.js (minimal and flexible), Fastify (high-performance), Koa (by the Express team, using generators), and NestJS (opinionated, TypeScript-first, Angular-inspired).

JavaScript frameworks and libraries dominate front-end web development. React, developed by Facebook (Meta) and released in 2013, uses a component-based architecture with unidirectional data flow and a virtual DOM for efficient UI updates. State management in large React applications typically uses Redux, Zustand, or React's built-in Context API. Angular, developed by Google, is a complete opinionated framework using TypeScript and a component-based architecture with two-way data binding. Vue.js is a progressive framework that can be adopted incrementally, with a gentle learning curve and excellent documentation. Next.js (React) and Nuxt.js (Vue) add server-side rendering and static site generation.

TypeScript, developed by Microsoft and released in 2012, is a superset of JavaScript that adds optional static typing, interfaces, enums, generics, and other features. TypeScript code is transpiled to plain JavaScript by the TypeScript compiler (tsc). TypeScript has become enormously popular for large-scale JavaScript applications because static typing catches many common bugs at compile time, improves IDE support with better autocompletion and refactoring tools, and serves as documentation. Most major JavaScript frameworks and libraries now either use TypeScript themselves or provide first-class TypeScript definitions.

JavaScript modules provide a way to organize code into separate files with explicit imports and exports. The ES module system (ESM), standardized in ES2015, uses import and export statements. The CommonJS module system (CJS), used by Node.js before ESM support, uses require() and module.exports. Bundlers like Webpack, Rollup, and esbuild process module dependencies and package code for deployment. Vite is a newer build tool that uses ES modules in development for extremely fast hot module replacement, falling back to Rollup for production builds.

Web APIs accessed through JavaScript in browsers include the Document Object Model (DOM) API for manipulating HTML and XML documents, the Fetch API for making HTTP requests, the Web Storage API (localStorage and sessionStorage) for client-side data persistence, the Web Workers API for running JavaScript in background threads, WebSockets for full-duplex communication, the Canvas API for 2D graphics, and WebGL for hardware-accelerated 3D graphics. Service Workers enable Progressive Web Apps (PWAs) to work offline by intercepting network requests and caching resources.

JavaScript's runtime environments have expanded beyond browsers and Node.js. Deno, created by Ryan Dahl (Node.js's creator) in 2020, provides a modern, secure JavaScript and TypeScript runtime with built-in TypeScript support, a permissions model that restricts file system and network access by default, and a standard library. Bun, released in 2022, is an extremely fast JavaScript runtime and toolkit built on JavaScriptCore (WebKit's engine) that aims to be a drop-in replacement for Node.js with dramatically better performance. Both Deno and Bun support npm packages.

WebAssembly (Wasm), standardized in 2019, is a binary instruction format that allows code written in languages such as C, C++, Rust, Go, and others to run in web browsers at near-native speed. JavaScript and WebAssembly can interoperate: JavaScript can call WebAssembly functions and vice versa. WebAssembly enables performance-critical applications like video editing, scientific computing, and game engines to run in the browser without plugins.
