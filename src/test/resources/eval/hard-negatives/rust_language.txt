Rust is a systems programming language focused on safety, performance, and concurrency. It was created by Graydon Hoare at Mozilla Research and first appeared publicly in 2010, with version 1.0 released in 2015. Rust is designed to provide memory safety without a garbage collector, using a compile-time ownership system to enforce safe memory access. The language targets the same performance envelope as C and C++ while eliminating entire classes of bugs common in those languages.

Rust's ownership system is its most distinctive feature. Every value in Rust has a single owner. When the owner goes out of scope, the value is dropped and its memory is freed. Ownership can be transferred (moved) to a new owner, after which the original binding is invalid. Values can also be borrowed, either as immutable references (any number simultaneously) or as a single mutable reference, but never both at the same time. These rules are enforced by the borrow checker at compile time, preventing data races and dangling pointer bugs without any runtime overhead.

Lifetimes are annotations that describe the scopes during which references are valid. The compiler infers lifetimes in most cases through lifetime elision rules, but complex scenarios require explicit annotations. Lifetimes ensure that references never outlive the data they refer to, eliminating use-after-free errors. While lifetimes can initially be challenging to understand, they give Rust programs strong safety guarantees verified at compile time.

Rust provides zero-cost abstractions: high-level constructs such as iterators, closures, and generics compile down to machine code as efficient as hand-written low-level code. Traits are Rust's mechanism for polymorphism. A trait defines a set of methods that a type can implement. Trait bounds constrain generic type parameters. Traits can also provide default method implementations. Rust uses static dispatch for generics (monomorphization) and dynamic dispatch for trait objects, giving developers control over the performance trade-off.

The Result and Option types are used for error handling and nullable values respectively. Result<T, E> represents either a success value (Ok(T)) or an error value (Err(E)). The ? operator propagates errors upward, making error-handling code concise. Option<T> represents either Some(T) or None, replacing null references. Both types are enums that the compiler requires to be exhaustively handled, preventing forgotten error checks.

Pattern matching in Rust is powerful and exhaustive. The match expression compares a value against a series of patterns and executes the first matching arm. The compiler verifies that all possible cases are handled. Patterns can destructure tuples, structs, enums, and references. The if let and while let constructs provide syntactic sugar for matching a single pattern.

Rust's concurrency model builds on ownership. The Send trait marks types safe to transfer between threads, and Sync marks types safe to reference from multiple threads. The compiler enforces these properties at compile time. The standard library provides Arc (atomically reference-counted shared ownership) and Mutex for shared mutable state across threads. The rayon crate provides parallel iterators for data parallelism.

Unsafe Rust allows code to perform operations the compiler cannot verify, such as dereferencing raw pointers, calling unsafe functions, or accessing mutable static variables. Unsafe blocks are explicit and narrow, limiting the scope of potentially unsound code. The goal is to contain unsafety to small, reviewable sections rather than applying it globally.

Cargo is Rust's build system and package manager. It handles dependencies declared in a Cargo.toml file, builds the project, runs tests, and generates documentation. Crates.io is the official package registry. Cargo integrates with the test runner and benchmark framework. The cargo clippy linter and cargo fmt formatter are standard tools in the Rust ecosystem.

Rust's macro system provides hygienic macros. Declarative macros (macro_rules!) match patterns and expand to code. Procedural macros operate on token streams and can derive trait implementations, define custom attributes, or implement function-like macros. The derive attribute automatically generates common trait implementations (Debug, Clone, PartialEq) from struct or enum definitions.

Async programming in Rust uses the async/await syntax. An async function returns a Future that must be polled to make progress. The Tokio runtime is the most widely used executor for running async Rust code, providing I/O, timers, and task scheduling. Unlike goroutines or threads, Rust futures are zero-overhead abstractions that compile to efficient state machines.

Rust is used in systems programming, embedded development, WebAssembly, game development, and increasingly in application software. Firefox uses Rust in its rendering engine components. The Linux kernel accepted Rust as a second implementation language in 2022. Cloudflare, Amazon, Google, and Microsoft have adopted Rust for performance-critical and security-sensitive components. The language has been voted the most loved programming language in Stack Overflow's developer survey every year from 2016 through the mid-2020s.

The Rust edition system allows breaking changes to be introduced while maintaining backward compatibility. New editions are opt-in per crate. Rust 2015, 2018, 2021, and later editions each introduced improvements to the language while allowing older code to continue compiling unchanged. The edition migration tool (cargo fix) automates most compatibility updates.

Rust's standard library is intentionally minimal, focusing on portable abstractions over operating system primitives. The no_std attribute allows Rust to target environments without an operating system, such as microcontrollers and BIOS code, using only the core library. The alloc crate adds heap allocation support without requiring a full standard library, enabling use in embedded contexts that have a memory allocator.
