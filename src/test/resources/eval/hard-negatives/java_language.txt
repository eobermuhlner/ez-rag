Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It was developed by James Gosling and colleagues at Sun Microsystems and released in 1995. Oracle Corporation acquired Sun Microsystems in 2010 and has since been the primary steward of the language.

Java is designed around the principle of write once, run anywhere (WORA). Java programs are compiled to platform-independent bytecode that runs on the Java Virtual Machine (JVM), which is available for virtually every operating system. This allows Java programs to run on any device with a compatible JVM installed without recompilation. The JVM specification is standardized, meaning multiple organizations can create compatible implementations.

Java is statically and strongly typed: variable types must be declared explicitly and are checked at compile time. This approach catches many common programming errors before the program is run, making Java well-suited for large-scale enterprise software development where reliability is critical. The Java compiler (javac) produces .class files containing bytecode, which the JVM then interprets or compiles to native machine code using just-in-time (JIT) compilation.

The Java platform includes the Java Development Kit (JDK) for development, which contains the compiler, debugger, and other tools, and the Java Runtime Environment (JRE) for running programs. Modern Java distributions since Java 9 combine these into a single JDK. Since Java 17 (2021) and Java 21 (2023), there are long-term support (LTS) releases that receive extended security updates, making them the preferred choice for enterprise deployments. Java releases now follow a six-month cadence, with a new version every spring and fall.

Java uses automatic garbage collection to manage memory. Programmers do not need to manually free memory; the garbage collector identifies objects that can no longer be reached by the program and deallocates them. The JVM includes several garbage collector implementations: Serial GC, Parallel GC, G1 GC (the default since Java 9), ZGC, and Shenandoah, each with different trade-offs in terms of throughput, latency, and memory footprint. Choosing the right GC and tuning its parameters is an important aspect of Java performance engineering.

Java's memory model divides storage into the heap and the stack. The heap stores all objects and is shared among all threads in a JVM process. The stack stores method call frames with local variables and is private to each thread. The distinction matters for concurrent programming: shared mutable heap objects require synchronization to avoid race conditions, while stack variables are inherently thread-safe. Java's volatile keyword and the java.util.concurrent.atomic package provide tools for safely sharing state across threads.

Java's generics, introduced in Java 5, allow classes and methods to be parameterized by type, enabling type-safe data structures and algorithms. However, generics in Java use type erasure: type parameters are removed at compile time and replaced with Object (or bounded types), meaning generic type information is not available at runtime. This is different from the approach in languages like C# or Kotlin. Common generic uses include collections (List<String>, Map<Integer, String>) and utility classes like Optional<T>.

Lambda expressions, introduced in Java 8, allow functional programming patterns by treating functions as first-class values. A lambda can be assigned to a variable or passed as an argument wherever a functional interface (an interface with exactly one abstract method) is expected. Java 8 also introduced the Stream API, which provides a declarative way to process collections of data through a pipeline of operations such as filter, map, reduce, collect, and sorted. Streams support lazy evaluation and can be parallelized for multi-core processing using parallelStream().

Java supports multithreading natively with the Thread class and the Runnable interface. The java.util.concurrent package, introduced in Java 5, provides higher-level concurrency utilities including ExecutorService for managing thread pools, Future and CompletableFuture for asynchronous computation, BlockingQueue for producer-consumer patterns, and various lock implementations. CompletableFuture in particular supports composing asynchronous operations in a non-blocking manner, analogous to promises in JavaScript.

The Java Collections Framework provides a unified architecture for representing and manipulating collections. The core interfaces are Collection (for groups of objects), List (ordered, allows duplicates), Set (no duplicates), Map (key-value pairs), and Queue. Common implementations include ArrayList (resizable array), LinkedList, HashSet (hash table), TreeSet (sorted), HashMap, and TreeMap. The Collections utility class provides static methods for sorting, searching, and synchronizing collections.

Java's module system, introduced in Java 9 as part of Project Jigsaw, allows code to be organized into named modules with explicit declarations of what packages they export and what modules they require. This improves encapsulation (internal packages can be truly hidden), reduces the attack surface for security vulnerabilities, enables smaller runtime images by including only needed modules, and speeds up startup time. The JDK itself was modularized in Java 9, replacing the monolithic rt.jar.

Records, introduced in Java 16, are a compact way to declare classes that are primarily carriers of immutable data. A record automatically generates a constructor, accessor methods, equals(), hashCode(), and toString() implementations based on its components. Sealed classes (Java 17) restrict which other classes or interfaces may extend or implement them, enabling exhaustive pattern matching. Text blocks (Java 15) allow multi-line string literals with automatic indentation stripping. Pattern matching for instanceof (Java 16) eliminates the need for explicit casting after an instanceof check.

Spring Framework is by far the most widely used Java application framework. It provides dependency injection (IoC container), aspect-oriented programming (AOP), transaction management, and integration with data access technologies (JPA, JDBC), web frameworks (Spring MVC, Spring WebFlux), and messaging systems. Spring Boot, a convention-over-configuration extension of Spring, makes it easy to create standalone, production-ready Spring applications with minimal setup. Spring Boot applications embed a web server (Tomcat by default) and can be deployed as self-contained executable JAR files.

Apache Maven and Gradle are the two dominant build tools for Java projects. Maven uses an XML-based project object model (pom.xml) and a convention-based project structure. Gradle uses a Groovy or Kotlin DSL (build.gradle or build.gradle.kts) and offers more flexibility and faster incremental builds through its build cache and daemon. Both tools handle dependency resolution from central repositories like Maven Central and JitPack.

Kotlin, developed by JetBrains and released in 2016, is a modern statically typed language that runs on the JVM (and can also compile to JavaScript or native code). Kotlin is fully interoperable with Java: Kotlin code can call Java code and vice versa without any adapter layer. Kotlin addresses several Java pain points: it has null safety built into its type system, more concise syntax (data classes, extension functions, smart casts), coroutines for asynchronous programming, and better support for functional programming. Google officially endorsed Kotlin as a first-class language for Android development in 2017, and it is now the preferred language for new Android projects.

JUnit is the standard unit testing framework for Java. JUnit 5 introduced a new programming model with annotations such as @Test, @BeforeEach, @AfterEach, and @ParameterizedTest. The Mockito library is widely used for creating mock objects in tests. The Spring Test module integrates Spring's IoC container into JUnit tests through the @SpringBootTest annotation. Test-driven development (TDD) is widely practiced in the Java community, where tests are written before implementation code to drive design.
