Ruby is a dynamic, open-source programming language with a focus on simplicity and productivity. It was designed by Yukihiro Matsumoto (known as Matz) and first released in 1995 in Japan, with the first English-language release following in 1996. The name Ruby was chosen as a reference to Perl, with ruby being a gemstone that comes after pearl.

Ruby's design philosophy is guided by the principle of least astonishment, also known as the principle of least surprise. This means the language should behave in ways that minimize surprise to experienced users, and that it should feel natural and intuitive. Matz has said that he designed Ruby to maximize programmer happiness rather than computer efficiency. The principle of least astonishment has influenced countless Ruby design decisions, from method naming conventions to how blocks and iterators work.

Ruby is a purely object-oriented language. Everything in Ruby is an object, including what other languages treat as primitive types: integers, floats, booleans, and even nil (Ruby's null equivalent). This means every value responds to methods: 42.times { puts "hello" } is valid Ruby that calls the times method on the integer 42. This uniformity distinguishes Ruby from languages like Java and Python where primitive types and object types are treated differently.

Ruby features a clean, readable syntax inspired by Perl, Ada, and Lisp. It uses end to close blocks instead of curly braces. Parentheses around method arguments are optional in many contexts. The language supports blocks, which are anonymous sections of code that can be passed to methods. Blocks are written using either do...end or curly braces, and they are one of Ruby's most distinctive features, enabling elegant iteration and callback patterns.

Modules in Ruby serve two purposes: they are used as namespaces to organize code, and as mixins to add shared behavior to classes. A module can be included in a class using the include keyword, which adds the module's methods as instance methods. The Enumerable module is one of Ruby's most powerful standard library modules: when included in a class that implements the each method, it provides dozens of collection methods including map, select, reject, find, sort_by, group_by, flat_map, reduce, and many more. The Comparable module provides comparison methods when a class implements the spaceship operator (<=>) for ordering.

Ruby's block mechanism underlies its powerful iterator pattern. The yield keyword inside a method body transfers control to a block passed to that method. Methods can accept explicit block parameters using the ampersand syntax (def process(&block)), which converts the block to a Proc object. This pattern is used throughout Ruby: Array#each yields each element, Hash#each yields key-value pairs, File.open yields a file handle and closes it when the block completes.

Procs and lambdas are both first-class function objects in Ruby, but they differ in how they handle return statements and argument checking. A Proc is created with Proc.new { } or proc { }. A lambda is created with lambda { } or the stabby lambda syntax ->. Lambdas enforce the number of arguments passed to them and return from the lambda only, while Procs are more lenient with arguments and return from the enclosing method. Understanding these differences is important for advanced Ruby metaprogramming.

Ruby on Rails, commonly called Rails, is the most influential web framework built with Ruby. Created by David Heinemeier Hansson (DHH) and released in 2004, Rails pioneered the Model-View-Controller (MVC) architecture and the convention over configuration principle, which means sensible defaults eliminate most boilerplate configuration. Rails follows the don't repeat yourself (DRY) principle. Active Record, Rails' object-relational mapping layer, maps database tables to Ruby classes automatically and provides a chainable query interface. Action Pack handles routing, controllers, and views. Rails popularized the idea that web applications could be built rapidly, and it heavily influenced later frameworks in many languages.

Ruby's metaprogramming capabilities are among the most powerful in any mainstream language. Programs can modify themselves at runtime: methods can be defined dynamically using define_method, classes can be reopened and modified at any point (monkey patching), and missing method calls are intercepted by method_missing. These features make DSLs (domain-specific languages) straightforward to create. Rails' migration syntax, RSpec's testing DSL, and countless configuration DSLs in the Ruby ecosystem are all enabled by metaprogramming.

Ruby's garbage collection uses a generational, incremental mark-and-sweep algorithm. Ruby 2.1 introduced generational GC, which distinguishes between young and old objects; young objects are collected more frequently because they tend to be short-lived. Ruby 2.2 improved incremental GC to reduce pause times. Ruby 3.x continues to improve the garbage collector's efficiency. The ObjectSpace module allows inspection and iteration over live objects, which is useful for debugging memory leaks.

The Ruby community strongly values test-driven development. RSpec is the most widely used BDD (behavior-driven development) testing framework, with a highly readable English-like syntax for describing expected behavior: describe, context, it, expect, and matchers like eq, include, and raise_error. Minitest ships with the Ruby standard library and supports both traditional assertion-style testing and RSpec-like spec syntax. Behavior-driven development with RSpec's readable syntax has influenced testing frameworks in many other languages.

RubyGems is the package management system for Ruby. Gems are packaged libraries and programs. Bundler manages gem dependencies for projects, ensuring consistent environments across different machines by reading from a Gemfile and generating a Gemfile.lock. The rubygems.org repository hosts hundreds of thousands of publicly available gems. Popular gems include Devise for authentication, Pundit for authorization, Sidekiq for background job processing, and Faraday for HTTP requests.

Ruby version management tools allow developers to install and switch between multiple Ruby versions on the same system. RVM (Ruby Version Manager) was the original tool and installs Ruby versions into a dedicated directory structure. rbenv is a lighter alternative that uses shims and the .ruby-version file to select the active version per directory. Chruby is an even simpler tool. These tools are essential because different projects may require different Ruby versions.

Ruby 3.0 (released on Christmas Day 2020, as has been traditional for Ruby releases) introduced several major features. Ractor (experimental) provides an actor-model concurrency primitive where Ractors run in parallel with no shared mutable state between them, working around the Global VM Lock (GVL, Ruby's equivalent of Python's GIL). Fiber Scheduler (also called the fiber scheduler interface) allows non-blocking I/O using fibers as the scheduling primitive. Pattern matching, introduced as experimental in Ruby 2.7, became stable in Ruby 3.0 and supports deconstruction of arrays and hashes.

JRuby is an implementation of Ruby that runs on the Java Virtual Machine, allowing Ruby programs to call Java libraries directly and to achieve true thread parallelism (since JRuby does not have a GVL). TruffleRuby is a high-performance Ruby implementation built on the GraalVM, using a JIT compiler for significantly faster execution of long-running programs. mruby is a lightweight implementation for embedded systems and mobile applications. These alternative implementations demonstrate the versatility of the Ruby language specification.

Sinatra is a minimal Ruby web framework that provides a simple DSL for defining routes and handling HTTP requests, without the full structure that Rails enforces. It is popular for small APIs, microservices, and as a learning tool for understanding web concepts. Hanami (formerly Lotus) is a more structured alternative to Rails that emphasizes clean architecture, separation of concerns, and immutability. Camping is an ultra-minimal framework famous for fitting into just 4 kilobytes.
